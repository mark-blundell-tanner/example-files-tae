# Leitfaden für Markdown-Dateien

## Überschriften

- Verwenden Sie eine h1-Überschrift (`#`) als Haupttitel der Datei
- Es sollte nur eine h1-Überschrift pro Datei geben
- Überspringen Sie keine Überschriftenebenen (z.B. nicht von h2 direkt zu h4)
- Setzen Sie ein Leerzeichen nach den `#`-Zeichen
- Verwenden Sie aussagekräftige, beschreibende Überschriften

## Formatierung

- Verwenden Sie **Fettdruck** für wichtige Begriffe und Hervorhebungen
- Verwenden Sie *Kursivschrift* für Betonungen oder Fremdwörter
- Verwenden Sie `Code-Formatierung` für technische Begriffe, Dateinamen, Befehle
- Vermeiden Sie GROSSBUCHSTABEN für Hervorhebungen

## Listen

- Verwenden Sie `-` für ungeordnete Listen
- Verwenden Sie `1.`, `2.`, etc. für geordnete Listen
- Fügen Sie ein Leerzeichen nach dem Aufzählungszeichen ein
- Halten Sie Listeneinträge konsistent formatiert
- Verwenden Sie bei verschachtelten Listen eine Einrückung von 2 oder 4 Leerzeichen

## Code-Blöcke

- Verwenden Sie dreifache Backticks für Code-Blöcke: ` ``` `
- Geben Sie die Programmiersprache nach den öffnenden Backticks an
- Beispiel: ` ```python `
- Stellen Sie sicher, dass Code-Blöcke geschlossen werden

## Links und Bilder

- Links: `[Beschreibender Text](URL)`
- Der Link-Text sollte aussagekräftig sein (nicht "hier klicken")
- Bilder: `![Alt-Text](pfad/zum/bild.png)`
- Alle Bilder benötigen einen aussagekräftigen Alt-Text

## Absätze und Zeilenumbrüche

- Trennen Sie Absätze durch eine Leerzeile
- Vermeiden Sie mehr als zwei aufeinanderfolgende Leerzeilen
- Vermeiden Sie Leerzeichen am Zeilenende
- Beenden Sie Dateien mit einer Leerzeile

## Sprache und Stil

- Schreiben Sie klar und verständlich
- Verwenden Sie kurze, prägnante Sätze
- Vermeiden Sie Fachjargon, wo es nicht notwendig ist
- Erklären Sie technische Begriffe beim ersten Vorkommen
- Verwenden Sie aktive statt passive Formulierungen
- Achten Sie auf korrekte deutsche Rechtschreibung und Grammatik

## Zeichensetzung

- Setzen Sie Kommas vor Relativsätzen (die, der, das)
- Verwenden Sie "dass" für Konjunktionen, "das" für Artikel/Pronomen
- Vermeiden Sie übermäßige Verwendung von Ausrufezeichen
- Verwenden Sie Gedankenstriche (–) statt Bindestriche (-) für Einschübe

## Tabellen

- Verwenden Sie `|` zur Trennung von Spalten
- Fügen Sie eine Kopfzeile mit `|---|---|` hinzu
- Halten Sie Tabellen übersichtlich und nicht zu breit
- Richten Sie Tabellenspalten für bessere Lesbarkeit aus (optional)

## Strukturierung

- Beginnen Sie mit einer Einleitung oder Übersicht
- Strukturieren Sie Inhalte logisch von allgemein zu spezifisch
- Verwenden Sie konsistente Nummerierung bei Anleitungen
- Fügen Sie bei Bedarf ein Inhaltsverzeichnis hinzu
- Beenden Sie mit weiterführenden Informationen oder Zusammenfassung

## Technische Aspekte

- Dateiendung: `.md`
- Zeichenkodierung: UTF-8
- Zeilenlänge: Idealerweise max. 80-120 Zeichen (bei Fließtext)
- Einrückung: Konsistent 2 oder 4 Leerzeichen (keine Tabs)

## Beispiele vermeiden

- Platzhalter-Links wie `#`, `TODO`, `xxx`
- Leere Überschriften oder Abschnitte
- Fehlerhafte oder unvollständige Syntax
- Nicht geschlossene Code-Blöcke oder Formatierungen
